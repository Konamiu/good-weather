# 《好天气》章节设计总集

> 玩法设计的唯一事实来源。事实依据见 `剧情总纲.md`，素材提示词见 `美术提示词全集.md`。
> 第1章的逐拍细节另见 `第一章设计文档.md`。

## 设计铁律

1. **玩法即回忆**：每章的核心操作必须是东当年真实做过的事，不加虚构桥段。
2. **前80%只准温暖，不准煽情**：眼泪靠三个手段——①重复过的操作最后被收回 ②场景复用变义 ③毫无防备时显影真实素材。
3. **显影系统**：所有真实素材第一眼是像素的，**长按才显影成真实原件**（照片/聊天截图/信）。绝不提示、不发光，靠她自己发现；发现数计入通关后的"抽屉"收藏。
4. **双视角**：开局选"他"或"她"。选"她"=以她视角走一遍被爱的故事；通关后出一行字"还有一些事，是你没看见的"，解锁"他"线（床铺两年/深夜备忘录/一个人的公交/网吧等她下班——全是她没看见的场景）。**眼泪落在二周目。**
5. **天气栏**：右上角常驻像素天气+日期计数（他数日子的习惯）。晴→雨→灰→雪→？随章节走，最终章揭示题眼。
6. **音乐**：《山楂树之恋》动机贯穿——第1章她第一次唱歌只给一小句；瑞安章明亮完整版；8月章单音慢放；床铺章八音盒循环；最终章全部停，放真实录音（若有）。

## 框架：备忘录（第0章，游戏的壳）

- 黑屏→手机亮起→备忘录APP列表，按日期排列，通关解锁。
- 列表混入1-2条"不是关卡"的条目：点开是东真实备忘录原文。
- 长按已通关条目=显影真实素材。
- 引擎现状：`src/scenes/memo.ts` 已实现列表+解锁；显影待接入真实素材文件。

## 结尾设计原则（预设，待剧本确认）

- 最后一章之后回到备忘录列表：所有条目变亮，多出最后一条今天日期的新备忘录——这是他现在写给她的话（真实的、此刻的）。
- 主题曲停 → 真实录音（若有）→ 黑屏一行字。
- 通关解锁"抽屉"：信的扫描件、聊天截图、照片、发现过的所有显影素材。
