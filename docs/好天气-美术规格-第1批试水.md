# 《好天气》像素美术规格 — 第1批（试水批）

给美术协作方（Codex）：请严格按以下规格产出。这是一个 PixiJS 竖屏像素叙事游戏，逻辑分辨率 **360×640**（游戏内按整数倍放大，所有素材按原始像素尺寸交付，不要预放大）。

## 全局硬性要求（每张图都必须遵守）

1. **PNG，透明背景**（背景图除外），**禁止抗锯齿**，禁止半透明羽化边缘——每个像素要么实色要么全透明
2. **禁止 AI 手绘感**：必须是严格网格对齐的像素画。建议用代码（node-canvas / sharp / 手写像素数组）程序化生成，保证网格干净
3. **统一调色板**（第1批限用以下16色，后续批次再扩展）：
   - 皮肤 `#f4d1ae` / 皮肤暗 `#d9a97c`
   - 头发黑 `#2b2028` / 发高光 `#4a3c46`
   - 男主蓝 `#4a6fa5` / 男主蓝暗 `#33507c`
   - 女主米白 `#efe6d5` / 女主裙红 `#c4554d`
   - 描边深棕 `#1f1a17`（所有角色统一1px外描边）
   - 夏日天 `#a8d8e8` / 路面灰 `#8a8580` / 树叶绿 `#6a994e`
   - 雨天灰蓝 `#5c6b7a` / 暖灯黄 `#f2cc8f` / 白 `#faf7f0` / 纯阴影 `#00000040`（仅阴影可用半透明）
4. **动画交付为横向条状 sprite sheet**：所有帧同尺寸从左到右排列，无间隙无边距
5. 命名：`{对象}_{动作}_{帧数}f_{单帧宽}x{单帧高}.png`，例 `boy_walk_4f_32x48.png`

## 验收标准
- 我方将直接以 `PIXI.Texture` + 等距切帧加载，**帧宽不一致/有边距即打回**
- 放大3倍后边缘必须锐利无模糊
- 角色5个文件之间比例、描边、色板必须一致（同一套人）

## 给Codex的建议做法
写一个 Node 脚本（sharp 或 node-canvas），把每个精灵定义成二维色号数组渲染成PNG——这样改色改帧都可复现。脚本和PNG一起交付。
