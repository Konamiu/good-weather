> 本版按专业像素美术规范重写（依据：tile-design/palettes/environment-design skill）。
> 视角：俯视3/4（宝可梦绿宝石/星露谷透视——建筑见正面+屋顶，物体投右下短影 #00000030 偏移2px）。
> 光源全项目统一：**左上方正午偏晒**。所有瓦片明暗方向必须一致，违者打回。

## 0. 项目色板 v2（色相偏移版，取代旧16色平涂三阶）

规则：**暗部色相转冷（偏蓝紫）、亮部转暖（偏黄）**，禁止纯明度渐变（死灰感）；阴影保留饱和度不许发灰。每个材质一条5阶ramp，全项目复用同一条ramp保证统一：

```
草地ramp（冷→暖旋转最大，夏天的浓绿）:
#1d3d30 → #2e6a3a → #4f8f42 → #7ab04e → #b8cf6b

水泥/路面ramp（冷底、仅高光翻暖）:
#3a3644 → #5c5866 → #8a8580 → #b0aa9c → #d8cfb4

红瓦/砖ramp:
#4a2530 → #7a3a38 → #a85544 → #c97a55 → #e8a878

树冠ramp（在草地ramp基础上加深一档使用后三阶+补最深一阶 #16302a）

皮肤ramp（沿用1.2批: #c9926b → #d9a97c → #f4d1ae，高光新增 #fbe4c0）

天空/水ramp: #4a6a9a → #6a92c0 → #a8d8e8 → #d0ecf4

夜间整体换色由引擎mood滤镜承担，素材只画白天版
```

角色主金色 `#f2cc8f` 升格为**保留色**：只允许出现在「可交互元素」上——趣事发现点闪光、目标箭头、冰心融化的光、她的发卡。场景装饰一律不得使用（专业术语reserve color，保证玩家一眼识别"能点的东西"）。

## 1. 地表瓦片集 v2 `tileset_ground_v2.png`（8列×8行，32×32）

三种地形 + **Wang 4-bit过渡件**（工程用自动拼接，件数按边+外角+内角=12件/地形对）：

**基础地形内部变体**（打破重复感，引擎按 70%/20%/10% 权重随机铺）：
- 行1：草地×3变体（纯/草簇/小白花）+ 土路×3变体（纯/车辙/杂草钻缝）+ 水泥×2变体（纯/裂缝）
- 行2：水泥变体（井盖/补丁/苔藓角/水渍）+ 人行道红砖人字纹 + 盲道×2 + 沙坑

**过渡件（本版核心增补）**：
- 行3：草→水泥 4边（N/E/S/W，草侵入水泥的锯齿状自然咬合边，不要直线切）
- 行4：草→水泥 4外角 + 4内角
- 行5：草→土路 4边 + 4外角（土路边缘要有草根蚕食感）
- 行6：草→土路 4内角 + 水泥→土路 4边（直边即可，人工物交界）
- 行7：树荫斑驳叠加片×4（半透明深绿 #16302a55，可叠任意地表；边缘做碎斑不要圆形）
- 行8：**磨损小路件×4**（草地上被踩出的浅土色痕迹：直/弯/端头/交叉——环境叙事用：从出租屋到小区A的路要用它铺，"他每天走的路"不用一句台词就讲完了）

## 2. 物件集修订要点（沿用批次D清单，追加以下规范）

- 每个物件底部投影统一右下2px（光源左上）
- 大物件（梧桐树2×3格、西瓜摊3×2格）单独成图交付，不进网格
- **新增**：出租屋→小区A的路边加一件「歪脖子老槐树+树下磨光的石墩」（2×2格）——大爷们坐出来的包浆，环境叙事件
- 宣传栏/招牌类文字区域一律留白（引擎贴字）

## 3. 地图构图规范（引擎侧执行，Codex出样张时也要遵守）

依据environment-design规范：
- **30-50%画面保持视觉安静**（大块草地/空路面），拒绝铺满装饰——密度要有节奏：小区门口密、路段疏、街心密
- 焦点规则：每屏最多1个视觉焦点（西瓜摊/棋摊/单元楼门），用亮度对比引导，不靠饱和度堆
- 玩家/NPC与地面的对比：地表ramp整体压低对比度一档，角色描边保持锐利——保证任何地表上角色都"跳出来"
- 樱花/落叶等粒子不用——蝉鸣的夏天靠光斑和热浪扭曲（引擎shader做），素材不出

## 4. 角色动画升级（依据animation-frames规范）

| 对象 | 现状 | 升级 |
|---|---|---|
| 男主/女主 walk 四方向 | 4帧 | **6帧**（4帧是NPC规格；主角步态要有重量：接触-下沉-推离-腾空-落地-恢复） |
| 男主 run_side | 4帧 | 6帧，加1帧前倾预备(anticipation) |
| 塞传单 stuff | 2帧 | **4帧**：预备(半蹲抬手)→塞入(冲击帧，传单变形1px)→收手→回正。冲击帧停留时间最短 |

| 猫/狗 | — | walk 4帧 + idle 2帧不变 |

交付名不变，帧数后缀更新（如 `boy_walk_down_6f_48x64.png`）。
