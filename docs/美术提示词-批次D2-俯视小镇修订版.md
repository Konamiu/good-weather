# 批次D2 · 俯视小镇生产规格（给 Codex 整份执行）

> 本文取代《美术提示词-批次D-俯视小镇.md》中的地表、角色动画和钩子 UI 规格；未在本文改写的 NPC 与建筑设定才回看旧批次D。
> 目标不是生成一张“像像素画的图”，而是生产可被 PixiJS 按固定网格切割、自动拼接、程序换色和重复生成的游戏素材。

## 0. 不可偏离的生产管线

1. **最终 PNG 必须用代码生成。**在 `tools/art/` 下编写 Node 脚本，复用 `pixel-core.mjs`；把轮廓、色块或瓦片定义为色号/色板索引数组，再渲染 PNG。不得用文生图直接出成品，不得把高分辨率插画缩小后伪装成像素图。
2. 生成脚本与 PNG 一起入库；脚本重复运行必须得到像素级相同的文件。输出统一放 `public/assets/`，不另建临时交付目录。
3. PNG 使用原始像素尺寸，禁止预放大；透明素材使用 RGBA，背景图不透明。禁止 JPEG、有损 WebP、抗锯齿、模糊、渐变、羽化和非整数缩放。
4. 外轮廓到透明区域必须是硬边：alpha 只能是 `00` 或 `ff`。仅地面投影允许 `#00000040`，仅树荫叠加片允许 `#16302a55`；二者边界也必须是硬像素，不做透明度渐变。
5. 统一光源为**左上方45°的夏日正午光**。亮面在左上，暗面与投影在右下；禁止沿轮廓均匀围一圈暗色的“枕头阴影”。
6. 描边统一使用1px选择性描边：角色、可交互物用相邻材质最深阶强化外轮廓；远景建筑和地表不套纯黑边。
7. 动画全部交付为**横向、等宽、无间隙、无裁边**的雪碧条。文件名中的尺寸是单帧尺寸，整图宽度必须等于“帧数×单帧宽度”。
8. 同时新增：
   - `tools/art/generate-d2.mjs`：统一生成本批素材；
   - `tools/art/verify-d2.mjs`：校验清单、PNG尺寸、色板、alpha、帧数、重复帧和瓦片接缝；
   - `package.json` 命令 `art:d2:generate`、`art:d2:verify`、`art:d2:refresh`。
9. 不要让旧的 `verify-characters-1.2.mjs` 用“目录中只能有10张48×64图”的规则误伤新素材；D2校验只检查本文清单，旧校验也应改为只核对自己的文件名集合。

### 0.1 先交一张工艺样张

完整生产前，先生成 `d2_craft_sample_320x180.png`，一张图同时放入：

- 3×3草地↔水泥拼接，必须包含直边、外角和内角；
- 一棵梧桐、一段建筑墙面、一个宣传栏；
- 东和楠各一帧正面站立，按真实游戏比例摆在地面上；
- 同一地块左半用错误的枕头阴影打叉，右半用正确左上光作对照。

样张只用于验收工艺，不进入游戏。必须在1×、4×、10×最近邻放大下检查；1×时轮廓和交互物仍应读得出来。

## 1. 项目色板 v2

### 1.1 色板纪律

- 暗部向冷色偏移，亮部向暖色偏移；禁止只把 RGB 同比例调暗/调亮。
- 同一种材质必须复用同一条 ramp，不得为单个物件临时造近似色。
- 单个小道具建议4—6个实色；单个角色建议8—12个实色；单张场景以约16色子色板为目标。新增颜色前先证明现有 ramp 无法表达。
- 下列色号约束的是**源 PNG**（素材只画白天版）。昼夜变化的引擎实现以现行 mood 滤镜为准（已上线）；day→dusk→night 色表映射（LUT）列为后期优化选项，是否切换由引擎侧（Claude）决策，素材侧不依赖任一方案。

### 1.2 材质 ramp（从最暗到最高光）

```text
草地/矮草： #1d3d30 → #2e6a3a → #4f8f42 → #7ab04e → #b8cf6b
树冠/深叶： #16302a → #245237 → #3b7740 → #62a148 → #9bc75c
水泥/路面： #3a3644 → #5c5866 → #8a8580 → #b0aa9c → #d8cfb4
土路/裸土： #3f2d32 → #65463b → #8b684a → #b18b5a → #d5b879
红瓦/红砖： #4a2530 → #7a3a38 → #a85544 → #c97a55 → #e8a878
皮肤：      #8f5f4a → #c9926b → #d9a97c → #f4d1ae → #fbe4c0
天空：      #4a6a9a → #5d84ad → #79acd0 → #a8d8e8 → #d0ecf4
水面：      #243f59 → #315f78 → #4f88a3 → #78b6c6 → #b6dce0
```

角色头发、蓝衣、米白衣和红裙继续使用1.2批既定 ramp，不重新换色。生成脚本必须把全部允许色集中声明并额外输出：

- `palette_v2_64x1.png`：每格1px的色板条；不足64格透明补齐；若实色超过64个则校验失败，必须先合并近似色；
- `palette_v2.json`：颜色名、hex、所属 ramp 与用途；透明色和两种半透明阴影作为例外项记录，不计入64个不透明色槽。

### 1.3 金色的真实语义

`#f2cc8f` 是**关系/记忆强调色**，可以出现在楠的发卡、标题分隔线、冰心暖光和重要记忆物上；它本身不等于“可点击”。

玩家可交互元素必须同时满足“金色提示＋至少一种动态语言”：1px跳动描边、上下1px浮动、两帧闪光或目标箭头。普通场景装饰禁止大面积使用金色，以免稀释重点。

## 2. 地表与自动拼接瓦片

统一网格32×32。基础层全部不透明；过渡层可以透明。相邻格边缘的首尾像素必须完全一致，不得靠引擎过滤掩盖接缝。

### 2.2 三套47块自动过渡图集

必须生成：

| 文件 | 前景 terrain | 背景 terrain | 整图 |
|---|---|---|---|
| `autotile_grass_on_cement_47t_256x192.png` | 草 | 水泥 | 8列×6行 |
| `autotile_grass_on_dirt_47t_256x192.png` | 草 | 土路 | 8列×6行 |
| `autotile_dirt_on_cement_47t_256x192.png` | 土路 | 水泥 | 8列×6行 |

这不是12块手工边角库，而是考虑N/NE/E/SE/S/SW/W/NW的47种合法组合。bit定义：`N=1, NE=2, E=4, SE=8, S=16, SW=32, W=64, NW=128`；只有相邻两条正交边都存在时，对角bit才允许存在。

图集按“合法 mask 升序”排布，8列×6行，最后一格透明：

```text
行1：0, 1, 4, 5, 7, 16, 17, 20
行2：21, 23, 28, 29, 31, 64, 65, 68
行3：69, 71, 80, 81, 84, 85, 87, 92
行4：93, 95, 112, 113, 116, 117, 119, 124
行5：125, 127, 193, 197, 199, 209, 213, 215
行6：221, 223, 241, 245, 247, 253, 255, 透明
```

绘制规则：

- 草侵入水泥：轮廓每8—14px出现1—2px草根或缺口，整体保持清楚，不能做随机锯齿噪声。
- 草侵入土路：边缘更柔软，允许2—4px零星草根；两种棕绿颜色交错最多一层。
- 土路侵入水泥：人为形成的硬边，轮廓较直，但每格至少有一处1—2px崩边。
- 所有内角、外角、T形和孤岛必须由脚本按同一组边缘模板组合，之后再手工像素修正；不得分别凭感觉画47张导致接缝漂移。
- `verify-d2.mjs` 需按mask生成至少100组随机3×3地图，确认无缺角、无透明洞、无越界色和边缘不连续。

### 2.3 地表叠加件 `tileset_ground_overlay_v2_256x64.png`

8列×2行，每格32×32，透明底：

1—4. 树荫斑驳片：疏/中/密/边缘，颜色仅 `#16302a55`，由3—10px不规则碎斑构成，禁止圆形柔光。

5—8. 磨损小路：横直、90°弯、端头、十字。浅土只占路径宽度8—12px，边缘被草蚕食；横直件必须可旋转生成竖直件。

9—12. 零散生活痕迹：粉笔线、落叶3片、瓶盖、干泥点；均不得承担碰撞。

13—16. 透明备用格，必须保持全透明。

### 4.2 六帧行走逐帧

完整循环是左右腿各走一步，没有腾空帧：

1. 右脚接触：右脚前、左脚后，身体位于中低位，左臂前摆；
2. 右脚通过：右脚经过身体正下方，双臂接近中线；
3. 右腿抬升：身体比帧1高1px，左脚仍接地，衣摆/短发反向滞后1px；
4. 左脚接触：与帧1相反，但不能简单整帧镜像，保留脸、发卡和衣服结构；
5. 左脚通过：与帧2相反；
6. 左腿抬升：与帧3相反，并确保帧6→帧1和帧3→帧4同样自然。

正面/背面脚尖横向错开2—3px表现前后；侧面步幅比正面宽4—6px。头部上下最多1px，脸部五官不得抖动，楠的短发只在帧3/6发梢偏移1px。

### 4.3 奔跑与塞传单

- 起跑两帧不进入循环：F1重心后移、膝盖压低2px；F2身体前倾3px、后脚蹬地，接下一张run F1。
- run循环6帧允许F2/F5各有一帧双脚离地，身体前倾始终保持；步幅约为walk的1.6倍，头发和衣角向后偏1—2px。F6必须自然接F1。
- 塞传单：F1半蹲抬手预备；F2手和传单插入门缝，传单前缘压扁1px，这是动作峰值；F3抽手但身体仍前倾；F4回正。四帧脚底不得横向漂移，门和门缝不画进角色PNG。

## 6. 物件扩充规则（P1/P2）

- 大物件独立成图：西瓜摊 `prop_melon_stall_96x64.png`、传达室 `prop_gatehouse_64x64.png`、石桌棋摊 `prop_chess_table_64x64.png`；不要塞进32px网格后裁断。
- 小物件统一放 `tileset_objects_v2_256x256.png`，8×8格。每个小物件必须完整位于单格，若超格就拆成独立文件。
- 宣传栏、招牌、手机卡片的文字区全部留白，中文由引擎绘制，禁止生成乱码像素字。
- 每个物件只保留2—4个生活痕迹，不要把“丰富”理解为撒满噪点。屏幕密度由地图组合控制。
- 交互物的金色动态提示单独成FX层，场景物件本体保持自然色，便于关闭提示和二周目复用。

## 8. 自动验收清单

`tools/art/verify-d2.mjs` 至少检查：

1. 文件清单齐全，文件名与大小写完全一致；
2. PNG真实尺寸与本文整图尺寸一致，动画宽度可被单帧宽整除；
3. 动画帧等宽、无间隙、无trim，每帧都不是完全重复；
4. 透明素材边缘无抗锯齿，alpha只出现 `00/40/55/ff`，且 `40/55` 只出现在规定的阴影文件或阴影区域；
5. 所有实色均来自 `palette_v2.json`，无近似脏色；
6. 32×32基础瓦片四边可重复，47块图集mask顺序正确，随机3×3地图无透明洞和缺角；
7. 角色脚底锚点固定、循环无root motion，walk没有双脚腾空，run只有规定帧腾空；
8. 左上光源一致，没有枕头阴影；角色和关键物件在1×缩放仍可辨认；
9. 用最近邻生成4×验收拼图，不改变原PNG；
10. 最终执行 `npm run art:d2:refresh` 与 `npm run build`，两者都必须通过。

出现任何一条失败，都先改色号数组/像素模板再重新生成，禁止直接在PNG上手修导致脚本与产物失配。
